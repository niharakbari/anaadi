//  db table

use anaadi;

CREATE TABLE design_images (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,

    original_filename VARCHAR(255) NOT NULL,

    stored_filename VARCHAR(255) NOT NULL UNIQUE,

    file_path VARCHAR(500) NOT NULL,

    file_size BIGINT UNSIGNED NOT NULL,

    mime_type VARCHAR(100) NOT NULL,

    image_width INT UNSIGNED NOT NULL,

    image_height INT UNSIGNED NOT NULL,

    uploaded_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,

    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
        ON UPDATE CURRENT_TIMESTAMP
);

ANAADI AI IMPLEMENTATION PLAN
Objective

Build an AI-powered jewellery image search system.

The dealer uploads a real jewellery photo, and the system returns the most visually similar catalogue design from approximately 2,000 stored design images.

The returned catalogue image already contains the design code (e.g., PS-0478-E), so there is no need to extract or store the code in the database during Phase 1.

Responsibilities
Node.js Backend

Responsible for:

Uploading catalogue images
Saving catalogue images to storage
Storing image metadata in MySQL
Uploading dealer search images
Calling the Python AI service
Returning search results

Node.js does not perform AI.

Python AI Service

Responsible for:

Loading the AI model
Image preprocessing
Generating image embeddings
Comparing embeddings
Returning Top-N matching image IDs

Python does not handle database logic.

AI Model

We will use an open-source image embedding model (such as CLIP/OpenCLIP or another strong vision embedding model).

Reason:

Free
No API cost
Runs locally
Designed for image similarity
Suitable for production deployment

We will not use ChatGPT, Gemini, or Claude Vision APIs for image search because they are not optimized for large-scale similarity retrieval.

Embeddings

Every catalogue image is converted into a numerical vector called an embedding.

Example:

Catalogue Image

↓

Embedding

↓

Stored in Vector Index

Dealer image:

Dealer Photo

↓

Embedding

↓

Compared with all catalogue embeddings

The closest embeddings represent the most visually similar designs.

Vector Search

Instead of comparing raw images, the AI compares embeddings.

Workflow:

Dealer Embedding

↓

Vector Search

↓

Top 5 Similar Catalogue Images

This is much faster and more accurate than comparing pixels.

Image Preprocessing

Preprocessing will be implemented during the AI phase.

Possible preprocessing:

resize image
normalize colors
remove unnecessary borders
crop specification table if testing shows it reduces accuracy
optionally separate multiple jewellery views into individual images

We will not implement preprocessing now.

We will first test using the original catalogue images.

Only if accuracy needs improvement will preprocessing be added.

Expected Accuracy

Dataset characteristics:

clean catalogue images
white background
high quality
approximately 2,000 images

Expected performance:

Top-1 match: approximately 75–90%
Top-5 match: approximately 90–95%

Actual performance will be validated using the client's real catalogue before deployment.
